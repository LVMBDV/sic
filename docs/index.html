Comments for your blog,
in one <script> tag.

sic is a small, opinionated comment service you can drop onto any site. One Node process, SQLite for storage, a single Docker image — and it stays out of your way.

Get started View on GitHub

What you get

Everything the embed needs, nothing it doesn't.

One-line embed

A single async <script> injects the widget. No build step on the host page.

Iframe-isolated

The widget renders in an iframe, so host CSS and JS can't break it — or read it.

OAuth login

GitHub and Google sign-in (PKCE for Google). No passwords to store, ever.

Up-vote reactions

Built in, one row per user per comment. Net counts come back with each thread.

Heuristic spam guard

Honeypot field, link-flood and caps-ratio checks, length cap. No third-party API.

First-party sessions

Signed-JWT cookies on the sic origin — HttpOnly, SameSite=Lax, Secure over HTTPS.

Tiny footprint

node:sqlite (no native build), Fastify, vanilla TS. No Redis, no Postgres, no bus.

One command to run

Ships as a single Docker image. docker compose up -d and you're live.

Quick start

Embed on a page, then run the service behind your reverse proxy.

1 · Embed on your page

<div id="sic-comments" data-thread="my-post-slug"></div>
<script src="https://comments.example.com/embed.js" async></script>

data-thread identifies the thread. Omit it and the canonical URL (or location.pathname) is used as a fallback.

2 · Run with Docker

cp .env.example .env   # set SIC_SESSION_SECRET, OAuth creds, allowed origins
docker compose up -d

The service listens on :6767. Put it behind Caddy, nginx, a Cloudflare Tunnel, or similar, and terminate TLS there.

Or run from source

cp .env.example .env
npm install
npm run dev            # backend :6767 + Vite :5173 (proxied)

Requires Node 24+ for stable node:sqlite.

Configuration

All via environment variables — see .env.example.

Variable Required Default Description
SIC_PUBLIC_URL ✅ — Public origin where sic is reachable. Used for OAuth redirects and cookie Secure.
SIC_SESSION_SECRET ✅ — 32+ chars of high-entropy random data. openssl rand -base64 32 is fine.
SIC_ALLOWED_ORIGINS ✅* — Comma-separated origins allowed to iframe the widget (CSP frame-ancestors + CORS).
SIC_BIND — 127.0.0.1:6767 Address to bind. Use 0.0.0.0:6767 in Docker.
DATABASE_URL — sic.db Path to the SQLite file. Defaults to /data/sic.db in Docker.
SIC_GITHUB_CLIENT_ID / _SECRET — — GitHub OAuth app credentials. Omit to disable GitHub login.
SIC_GOOGLE_CLIENT_ID / _SECRET — — Google OAuth credentials. Omit to disable Google login.
LOG_LEVEL — info trace, debug, info, warn, error.

* Required in practice — without it, no third-party site can iframe the widget.

OAuth callbacks

Register an app with each provider and set the callback URL to:

GitHub:  ${SIC_PUBLIC_URL}/auth/github/callback
Google:  ${SIC_PUBLIC_URL}/auth/google/callback

How it fits together

The widget is served from the sic origin, so session cookies stay first-party. Host sites never see them. 

host page ──<script>── embed.js ──injects──▶ <iframe src="…/embed?thread=…">
                                                      │
                                                      ▼
                                              sic backend (:6767)
                                              ├── /embed              (widget HTML)
                                              ├── /embed.js           (loader)
                                              ├── /api/me
                                              ├── /api/threads/:slug/comments
                                              ├── /api/comments/:id/reactions/:kind
                                              └── /auth/{github,google}/{login,callback}
                                                      │
                                                      ▼
                                                SQLite (WAL)

The widget posts sic:resize messages to the parent so the iframe height tracks its content. Sessions are signed JWTs with no server-side store — revoke by rotating SIC_SESSION_SECRET.
